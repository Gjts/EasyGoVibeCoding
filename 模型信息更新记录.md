## 自动化更新机制

```text
Cloudflare Worker (model-updater)
└── Cron Trigger: 0 */6 * * *        每 6 小时（UTC 0/6/12/18 点）
    ├── 调用 LLM 联网搜索 API         默认 Perplexity sonar-pro，可切换
    ├── zod 校验 ModelsPayloadSchema  失败重试 1 次，仍失败保留旧值
    └── 写入共享 KV                   models:latest / models:previous / models:history:YYYY-MM

Cloudflare Pages Function
└── GET /api/models                   从 KV 读取，ETag 协商缓存

前端（Next.js 静态导出）
├── data/models.json                  构建时种子数据（fallback）
├── lib/models.ts                     客户端 fetch + 失败回退 + zod 再校验
└── components/home/latest-models-panel.tsx  首页「最新模型发布追踪」面板
```

## 维护规则

1. 动态模型内容至少每周人工抽查一次，重点核对 OpenAI、Anthropic、Google 三类高曝光入口。
2. Worker 写入 KV 后，应抽查 `/api/models` 返回的 `source`、`updatedAt` 和首页徽章，确认当前展示来自 API 还是静态种子数据。
3. 对没有官方模型页或 API 文档支撑的条目，不应写成“最新旗舰”或“已开放 API”；只能标注为“待核验”“私有预览”或从展示中移除。
4. 新闻稿可作为背景材料，但模型 ID、上下文窗口、价格、稳定性和可用工具以官方模型页/API 文档为准。
