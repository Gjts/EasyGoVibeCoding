## 自动化更新机制（2026-04-20 上线）

从 2026 年 4 月起，模型动态信息接入自动化抓取管线，不再完全依赖手工维护。

### 架构

```
Cloudflare Worker (model-updater)
   └─ Cron Trigger: 0 */6 * * *        每 6 小时（UTC 0/6/12/18 点）
        ├─ 调用 LLM 联网搜索 API         默认 Perplexity sonar-pro，可切换
        ├─ zod 校验 ModelsPayloadSchema  失败重试 1 次，仍失败保留旧值
        └─ 写入共享 KV                   models:latest / models:previous / models:history:YYYY-MM

Cloudflare Pages Function
   └─ GET /api/models                   从 KV 读取，ETag 协商缓存

前端（Next.js 静态导出）
   ├─ data/models.json                  构建时种子数据（fallback）
   ├─ lib/models.ts                     客户端 fetch + 失败回退 + zod 再校验
   └─ components/home/latest-models-panel.tsx  首页「最新模型发布追踪」面板
```

### 涉及文件

- Worker 工程：`src/backend/model-updater/`（含 `wrangler.toml` / `providers/*` / `kv.ts` / `updater.ts` / `README.md`）
- 共享 Schema：`src/frontend/EasyGoVibeCoding/lib/model-schema.ts`
- Pages Function：`src/frontend/EasyGoVibeCoding/functions/api/models.ts`
- 种子数据：`src/frontend/EasyGoVibeCoding/data/models.json`
- 前端消费：
  - `src/frontend/EasyGoVibeCoding/lib/models.ts`
  - `src/frontend/EasyGoVibeCoding/components/home/latest-models-panel.tsx`
  - `src/frontend/EasyGoVibeCoding/components/interactive/model-tier-list.tsx`
  - `src/frontend/EasyGoVibeCoding/app/ecosystem/page.tsx`
  - `src/frontend/EasyGoVibeCoding/app/tools/selection/page.tsx`

### 运维动作

- **首次部署** 见 `src/backend/model-updater/README.md`
- **手动触发** `curl -X POST https://model-updater.<account>.workers.dev/__run`
- **查看日志** `cd src/backend/model-updater && pnpm tail`
- **回滚** 把 KV 里 `models:previous` 写回 `models:latest`

### 数据更新频率

- Worker 定时任务：每 6 小时
- Pages Function 响应缓存：`Cache-Control: public, max-age=300`
- 前端展示：页面加载后立即拉取 `/api/models`，失败回退到构建时种子数据

### 降级链

```
运行时 API 正常   ─▶  展示 KV 实时数据（徽章「实时 API」）
API 失败 / 4xx   ─▶  回退到 data/models.json（徽章「静态种子」）
种子 JSON 损坏   ─▶  构建失败（zod parseSeedOrThrow 会在 build 阶段抛错）
```
