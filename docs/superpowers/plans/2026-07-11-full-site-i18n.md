# Full-Site Japanese, English, French, and German Translation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Generate, build, and verify complete Japanese, English, French, and German static versions of every Chinese academy page without translating at request time.

**Architecture:** Keep Chinese source files authoritative. Extract translatable TS/TSX/JSON literals with the TypeScript compiler, translate them in cached batches, apply each locale catalog to a disposable source copy, and run one independent Next.js static build per locale. Merge target builds below `/ja/academy`, `/en/academy`, `/fr/academy`, and `/de/academy`; the existing Japanese sales site remains at `/ja`.

**Tech Stack:** Next.js 16 App Router, React 19, TypeScript compiler API, Node.js built-in `fetch`/`node:test`/`crypto`/`fs`, Cloudflare Pages static export, OpenAI-compatible Chat Completions API.

## Global Constraints

- The canonical editable source remains Simplified Chinese.
- Target locales are exactly `ja`, `en`, `fr`, and `de`.
- The existing Japanese sales pages under `/ja` remain unchanged; Japanese academy routes use `/ja/academy`.
- Translation API calls run only from explicit generation scripts, never in the browser, Next.js render path, or Cloudflare request path.
- `TRANSLATION_API_KEY` remains server-only in ignored `.env.local`; no `NEXT_PUBLIC_` secret is allowed.
- No new npm dependency is added; use the already-installed `typescript` package and Node built-ins.
- Every localized build must be a real static export with matching localized client chunks so React hydration cannot restore Chinese text.
- Existing Chinese URLs remain unchanged.
- Generated translation catalogs are committed; disposable localized source trees and API caches remain under ignored `.cache/`.
- Source placeholders, code expressions, URLs, product names, Markdown, and HTML tags must survive translation.
- Missing keys, unknown keys, dropped placeholders, residual non-whitelisted Han text, broken links, or failed builds are release blockers.

- [ ] **Step 3: Implement TypeScript AST extraction**

```js
export function extractTranslationUnitsFromText({file, source}) {
  const sourceFile = ts.createSourceFile(
    file,
    source,
    ts.ScriptTarget.Latest,
    true,
    file.endsWith(".tsx") ? ts.ScriptKind.TSX : ts.ScriptKind.TS,
  )
  const units = []
  const visit = (node) => {
    if (ts.isImportDeclaration(node) || ts.isExportDeclaration(node)) {
      return
    }
    if (ts.isPropertyAssignment(node) && ts.isStringLiteral(node.name)) {
      visit(node.initializer)
      return
    }
    if (ts.isJsxText(node) && HAN_PATTERN.test(node.text)) {
      units.push(createLiteralUnit({file, node, source, kind: "jsx-text"}))
    } else if (
      (ts.isStringLiteral(node) || ts.isNoSubstitutionTemplateLiteral(node)) &&
      HAN_PATTERN.test(node.text)
    ) {
      units.push(createLiteralUnit({file, node, source, kind: "literal"}))
    } else if (ts.isTemplateExpression(node) && HAN_PATTERN.test(node.getText(sourceFile))) {
      units.push(createTemplateUnit({file, node, sourceFile}))
      return
    }
    ts.forEachChild(node, visit)
  }
  visit(sourceFile)
  return units
}
```

export const TARGET_LOCALES = ["ja", "en", "fr", "de"] as const

export function stripLocaleBasePath(pathname: string): string {

export function localizedAcademyPath(locale: string, canonicalPath: string): string {
  const path = canonicalPath === "/" ? "" : canonicalPath
  return locale === "zh-CN" ? canonicalPath : `/${locale}/academy${path}`
}
